### 1) 三个排序算法（Algorithm A / B / C）性质对比

### 题目概述
给出三段 `sort` 代码（Algorithm A/B/C），完成表格，分别填写：
- 是否正确 Correct (Y/N)
- 最坏时间复杂度 Time Complexity
- 最坏空间复杂度 Space Complexity
- 是否原地 In-Place (Y/N)
- 是否破坏性 Destructive (Y/N)

### 最终答案表

| 项目 | Alg. A | Alg. B | Alg. C |
|---|---|---|---|
| Correct (Y/N) | **Y** | **Y** | **Y** |
| Time Complexity（最坏） | **O(n²)** | **O(n log n)** | **O(n²)** |
| Space Complexity（最坏） | **O(n²)** | **O(n)** | **O(n)** |
| In-Place (Y/N) | **N** | **N** | **Y** |
| Destructive (Y/N) | **N** | **Y** | **Y** |

### 解析要点（简洁版）
- **Algorithm A：列表推导版 QuickSort**
  - 每次递归用列表推导生成 `left/middle/right`（新列表），不是原地。
  - 最坏情况划分极不均衡（递归深度 ~ n），每层还会创建接近 n 的新列表，**最坏额外空间可达 O(n²)**。
  - 返回新数组，不修改原数组 ⇒ **非 destructive**。
- **Algorithm B：MergeSort（切片 + 写回）**
  - 递归分成 `l_half/r_half` 并合并写回 `arr`，整体最坏 **O(n log n)**。
  - 由于切片/辅助数组需要额外空间 **O(n)**，且会修改原数组 ⇒ **destructive，但非 in-place**。
- **Algorithm C：递归版 Insertion Sort**
  - 递归排序前 n-1 个元素，再把最后元素插入到正确位置；最坏 **O(n²)**。
  - 只在原数组内部移动元素，额外主要是递归栈 **O(n)** ⇒ **in-place 且 destructive**。

### 2) 题：`max_heapify` 找 bug + 写 `heapsort`
### (a) 题目
给出如下 `max_heapify` 实现，但它不能正确维护最大堆性质：
- 指出 bug 与对应行号
- 修改**最多两行**修复

### 答案：bug 行号与修复

**Bug 1：比较方向写反（像最小堆）**
- 问题行：**第 6 行**
- 原代码：
  ```python
  if left < n and arr[left] < arr[l]:
  ```
- 修复为（最大堆应选更大的）：
  ```
  if left < n and arr[left] > arr[l]:
  ```
**Bug 2：交换写错（右边重复 arr[l]，导致值丢失）**

- 问题行：第 13 行

- 原代码：
```
arr[i], arr[l] = arr[l], arr[l]
```
- 修复为标准交换：
```
arr[i], arr[l] = arr[l], arr[i]
```

### 3) 堆（Heap）选择题（i~v）

![1765093814961](image/Exam/1765093814961.png)
![1765093825400](image/Exam/1765093825400.png)
### 题目（概述）
每题从 A~E 选最佳答案：

i. 在 size 为 n 的二叉最大堆插入一个元素，时间复杂度？  
ii. 最大堆插入操作过程中发生什么？  
iii. heapify 函数在二叉堆中的目的？  
iv. 用 heapify 在 n 个元素上建堆的时间复杂度？  
v. Heapsort 的空间复杂度？

### 最终答案（字母）
- i：**B**
- ii：**B**
- iii：**B**
- iv：**A**
- v：**C**

### 解析要点（简洁版）
- i：插入后最多上浮到根，层数是堆高 **O(log n)** ⇒ **B**
- ii：新元素先放到数组末尾（最后位置），再向上堆化（上浮） ⇒ **B**
- iii：heapify 用于在某个结点处**恢复子树的堆性质** ⇒ **B**
- iv：自底向上 heapify 建堆，总体摊还是 **O(n)** ⇒ **A**
- v：Heapsort 在数组上原地完成（不计递归栈）额外空间 **O(1)** ⇒ **C**
